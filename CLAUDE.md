# ケンとハナの食卓バトル - AI制作プロジェクト

## このプロジェクトについて
毎回1つの料理を取り上げ、栄養素・歴史・文化的背景をもとに友人2人が口論する料理雑談ラジオ番組の台本を制作する。

**番組コンセプト**：料理にまつわるデータと食欲がぶつかるバトルラジオ。
**リスナー**：食に興味がある20〜40代。
**テーマ方針**：料理1品を深掘り。歴史・栄養・文化・食べ方など多角的に扱う。

## フォルダ構成
- characters/profile.md  → キャラ設定（必ず参照）
- knowledge/             → リサーチ結果
- scripts/               → 生成済み台本・音声ファイル
- workflow/              → 各作業の手順書
- tts.py                 → 音声生成スクリプト
- .gitignore             → __pycache__ など除外設定

## 作業フロー（完全自動）
テーマを受け取ったら、途中で止まらず以下を一気に実行すること。承認待ちはしない：

1. characters/profile.md を読み込む
2. workflow/research.md の手順でリサーチ（ウェブ検索で情報収集し、ナレッジ保存）
3. workflow/scriptwriting.md の手順で構成を内部で確定（提示・承認待ちはしない）
4. そのまま台本生成
5. workflow/review.md の手順でチェック（不合格なら自動で修正して再チェック）
6. 全項目合格したら scripts/ に保存
7. workflow/audio.md の手順で音声ファイルを生成（VOICEVOXはtts.pyが自動起動する）
8. workflow/publish.md の手順で、その回の生成物（台本・音声・ナレッジ）をGitHubへcommit & push

完了後に、生成した台本・音声ファイルのパスと要点、pushの結果をまとめて報告する。

## 絶対ルール
- リサーチはウェブ検索ベース（検索エンジンAPIを使用）
- 出典は「ウェブ検索結果より」と正直に記載する（直接アクセスしていないサイトを出典として記載しない）
- キャラの口調はprofile.mdに従うこと
- フローは承認待ちをせず音声生成・GitHubへのpushまで自動で進めること
- GitHubへのpushは、その回の生成物（台本・音声・ナレッジ）のみを対象にする（git add -A は使わない）
- チェックリストをパスしないとファイル保存しないこと

## ファイル命名規則
scripts/YYYYMMDD_ep###_テーマ一言.md
例：scripts/20260705_ep009_ラーメン体にいいの悪いの.md

台本を修正する場合は上書き禁止。末尾にバージョン番号を付けた新ファイルとして保存する。
- 初回：scripts/YYYYMMDD_ep###_テーマ一言.md
- 修正1回目：scripts/YYYYMMDD_ep###_テーマ一言v2.md
- 修正2回目：scripts/YYYYMMDD_ep###_テーマ一言v3.md
音声ファイルも同様（tts.pyが自動で同名.wavを生成するため自然に対応する）。
