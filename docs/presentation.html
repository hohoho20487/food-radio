TOOL — Superpowers
設計を加速するSuperpowersスキル
🔌Superpowers とは

Claude Code に追加できるスキルプラグインのフレームワーク。
複雑なワークフローを再利用可能な「スキル」として定義し、呼び出すだけで専門的な手順を実行する。

1
brainstormingスキル：Q&A対話で要件を整理し、設計案を提示・承認後に設計書を自動生成する
2
設計書は docs/superpowers/specs/ へ自動保存・git commit
3
他にも code-review・writing-plans など多数のスキルが利用可能
🎙️今回の使いかた
Q&A 対話
→
設計案を提示
→
承認
→
設計書を保存
使ったスキル：superpowers:brainstorming
生成された設計書：

TECH — VOICEVOX連携
音声生成の仕組み
⚙️tts.py の処理フロー
1
台本 .md を読み込み、キャラ名：「セリフ」 の行をすべて抽出
2
VOICEVOXが起動していなければ exe を自動起動し、APIが応答するまで待機
3
セリフごとに HTTP POSTで2回リクエスト（音声クエリ生成 → 音声合成）
4
全セリフの WAV をメモリ上で結合して1ファイルに書き出す
💻APIの呼び出しコード
# ① 音声クエリを生成
query = requests.post(
  "http://localhost:50021/audio_query",
  params={"text": text, "speaker": 82}
)

# ② WAVデータを合成
synthesis = requests.post(
  "http://localhost:50021/synthesis",
  params={"speaker": 82},
  json=query.json()
)
# speaker ID で声を切り替え
# ケン: 82 / ハナ: 65
✅ Claude Code が python tts.py 台本.md を実行するだけ。VOICEVOX の起動・音声生成・ファイル保存まで Python スクリプトが全部やる。
TECH — GitHub連携
自動バックアップの仕組み
💾GitHub をバックアップに使う理由
①
履歴管理：どのテーマで何を作ったか、いつでも過去に戻れる
②
ファイルセット管理：台本・音声・調査メモをエピソード単位でまとめて保存
③
安全性：ファイル単位で対象を管理し、意図しないファイルの混入を防ぐ
④
誤保存防止：設定ファイルや他エピソードは対象外。パス指定で厳密に管理
⑤
認証：GitHub の接続情報は Windows 資格情報マネージャー で管理。gh auth login で一度認証すれば、以降の push は自動で通る
💻Claude が実行する git コマンド
# その回のファイルだけを指定してステージング
git add "scripts/20260704_ep009_ラーメン.md" \
         "scripts/20260704_ep009_ラーメン.wav" \
         "knowledge/ramen_nutrition.md"

# コミット
git commit -m "Add ep009 script, audio, and knowledge: ラーメン"

# プッシュ（force は絶対に使わない）
git push origin main
💡 git add . は使わない。パス指定で対象ファイルだけをコミットするルールを Claude Code に事前定義しておくことで、意図しないファイルの混入を防ぐ。

今日からできること

テーマだけ渡す
調査・台本・音声・バックアップまで人間の介入ゼロ。「作る」コストを限りなく下げられる。

AIと一緒に設計する
